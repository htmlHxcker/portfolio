Why did I build this

This was the first project I ever built with React and it was a huge shift in terms of mindset compared to vanilla JS and any other library I had ever tried. I was basically lost in the React jungle and this helped me make sense of a lot of things. I experienced props drilling first hand and found a solution to it, I also went under the hood by using just webpack instead of create-react-app.

Why I chose this tech stack

Although only vanilla JavaScript is allowed for building Chrome extensions, after preparing a list of features I wanted the extension to have, it wasn't long before I realized this was a job better suited to React.
A lot of the UI Components were reusable, managing the state of the app and keeping that state handy for when the user modified the app, all these were features that made React a natural choice for me.
I decided to go with Redux for state management because props drilling was becoming a serious issue. The state itself was quite complex and I figured React's useContext hook may not cut it.

Problems and Improvements.

This was my first experience with the "it works on my machine" phenomenom. Due to the fact that the Chrome extension API was very different from the native Browser API, I had a very convoluted development process.

I would work on a new feature and have no idea if my code worked, I'd have to run the build step, reload the extension in the browser and only then would my changes be reflected offering me a rather humbling glimpse into life before hot-reloading. üòÖ
